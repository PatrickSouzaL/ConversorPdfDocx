## 1. Lógica / Código atual

Projeto de rotina de backend que converte em lote PDFs (`./pdf`) para DOCX (`./docx`).

### Arquitetura modular (`main.py`)

| Função | Responsabilidade |
| --- | --- |
| `discover_pdfs()` | Varre a origem e aplica filtro estrito `.pdf` (case-insensitive), não recursivo. |
| `target_path_for()` | Resolve o destino `.docx` mantendo o nome de origem. |
| `_plan_jobs()` | Separa o que converter do que ignorar (idempotência). |
| `convert_single()` | Converte **um** arquivo em processo worker isolado (fail-safe + limpeza de `.docx` parcial). |
| `run_batch()` | Orquestrador: fila, `ProcessPoolExecutor`, progresso e log. |
| `summarize()` / `_log_result()` | Agregação e log por status. |
| `build_parser()` / `main()` | CLI (`argparse`) e ponto de entrada. |

### CLI

```powershell
python main.py                 # ./pdf -> ./docx (defaults)
python main.py --workers 4     # nº de processos
python main.py --force         # reprocessa mesmo se .docx existir
python main.py --src pdf --dst docx --verbose
```

Flags: `--src`, `--dst`, `--workers` (padrão = nº de CPUs, limitado ao nº de arquivos), `--force`, `--verbose`.
Saída: log em console + `conversao.log`. Exit code `1` se houver qualquer falha (para CI).

### Stack

- Python 3.10+ (validado em **3.14.6**).
- `pdf2docx==0.5.13` (traz `PyMuPDF 1.28.0` transitivamente).
- Instalação: `pip install -r requirements.txt`.

### Regras de negócio implementadas

- **Fail-safe:** falha em um PDF é capturada, registrada e não interrompe o lote (isolamento por processo).
- **Idempotência:** nome preservado (`.pdf`→`.docx`) + skip de arquivos já convertidos (salvo `--force`).
- **Filtro estrito:** apenas `.pdf` (case-insensitive); demais arquivos ignorados.

## 2. Decisões técnicas firmadas nesta sessão

1. **Engine `pdf2docx`** (sobre extração de texto puro): preserva layout, tabelas e imagens, entregando DOCX editável.
2. **`ProcessPoolExecutor` (processos, não threads):** conversão é CPU-bound e a engine libera pouco o GIL; processos dão paralelismo real **e** isolam crashes de baixo nível (segfault não derruba o lote).
3. **`pathlib`** para resolução de caminhos multiplataforma.
4. **Limpeza de artefato parcial:** `.docx` incompleto é removido em caso de falha.
5. **Versionamento:** PDFs/DOCXs (documentos internos) e `.claude`/`.agents` ficam fora do Git (`.gitignore`); pastas preservadas via `.gitkeep`. Commits sem trailer *Co-Authored-By*.
6. **Escopo v1.0.0:** OCR fora de escopo — PDFs escaneados/baseados em imagem têm as imagens embutidas no DOCX, mas **sem texto pesquisável**.
